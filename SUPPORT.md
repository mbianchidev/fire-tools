# Support

Thank you for using Fire Tools! This document explains how to get help with the project.

## Getting Help

### GitHub Issues (Primary Support Channel)

The primary way to get support is by **opening an issue** on GitHub:

üëâ [Open an issue](https://github.com/mbianchidev/fire-tools/issues/new)

This allows the community to:
- See and benefit from questions and answers
- Search for similar issues before asking
- Track progress on bug fixes and feature requests

### What to Include in Your Issue

To help us help you quickly, please include:

1. **A clear title** - Summarize your question or problem
2. **Detailed description** - Explain what you're trying to do
3. **Steps to reproduce** (for bugs) - Exact steps to recreate the issue
4. **Expected vs actual behavior** - What should happen vs what happens
5. **Screenshots** - Visual aids help immensely for UI issues
6. **Environment details**:
   - Browser and version (e.g., Chrome 120, Firefox 121)
   - Operating system (e.g., Windows 11, macOS 14, Ubuntu 22.04)
   - Node.js version (if relevant for development)

## Types of Support

### Bug Reports

Found a bug? Please report it! Include:
- Description of the bug
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots or error messages
- Your browser and OS

### Feature Requests

Have an idea for a new feature? We'd love to hear it! Include:
- Description of the feature
- Why it would be useful
- How you envision it working
- Any examples from other tools

### Usage Questions

Need help using Fire Tools? Ask away! We're here to help you:
- Understand how features work
- Configure your FIRE calculations
- Export and import your data
- Troubleshoot issues

### Documentation Issues

Found a typo or unclear documentation? Let us know:
- What documentation is unclear
- Where you found it
- How it could be improved

## Response Time

- **Initial response**: We aim to respond to issues within **3-5 business days**
- **Resolution time**: Varies based on complexity and priority
- **Community help**: Other users may respond sooner!

Please be patient - this is an open source project maintained by volunteers.
